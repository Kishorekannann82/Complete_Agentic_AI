What is a Data Loader?

A Data Loader is responsible for loading data from a dataset and feeding it to a machine learning model efficiently. Instead of loading the entire dataset into memory, it loads data in small batches, making training faster and memory efficient.

Why Do We Need a Data Loader?

  • Loads data in batches
  • Shuffles training data
  • Supports parallel loading using multiple workers
  • Improves GPU utilization
  • Reduces memory usage

Workflow

  1. Dataset stores all samples.
  2. DataLoader reads samples.
  3. Creates mini-batches.
  4. Optionally shuffles data.
  5. Feeds batches to the model.
Tip: Mini-batch training is faster and more stable than processing the entire dataset at once.

PyTorch Example

from torch.utils.data import DataLoader
from torchvision.datasets import MNIST
from torchvision import transforms

dataset = MNIST(
    root="data",
    train=True,
    download=True,
    transform=transforms.ToTensor()
)

loader = DataLoader(
    dataset,
    batch_size=32,
    shuffle=True
)

for images, labels in loader:
    print(images.shape, labels.shape)
    break

Important Parameters

Parameter Description
batch_size Number of samples in one batch.
shuffle Randomly shuffles data every epoch.
num_workers Loads data using multiple CPU workers.
drop_last Drops the last incomplete batch.
pin_memory Speeds up GPU data transfer.

Summary

  • Dataset stores data.
  • DataLoader serves batches.
  • Batching improves speed.
  • Shuffling improves generalization.
  • Essential for deep learning training.
